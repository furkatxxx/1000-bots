# Система обратной связи — детальный проект

## Утверждено: 2026-03-22

## Концепция
Заменить бесполезные звёзды (1-5) на **причины** отклонения/одобрения.
Из накопленных причин строится "профиль вкуса" → калибрует промпт генерации.

## Причины отклонения (rejectReason)
| Код | Кнопка | Что узнаёт Claude |
|-----|--------|-------------------|
| `vague` | Абстрактно | Нет конкретной боли → генерить конкретнее |
| `crowded` | Уже есть решения | Рынок занят → искать незанятые ниши |
| `not_my_profile` | Не моё | Нужен найм/лицензии/физика → жёстче фильтровать |
| `bad_economics` | Не сойдётся | CAC > LTV, рынок мелкий → считать экономику |
| `boring` | Не цепляет | Субъективно, но если паттерн — учтём |

## Причины одобрения (likeReason) — мультиселект
| Код | Кнопка | Что узнаёт Claude |
|-----|--------|-------------------|
| `real_pain` | Боль реальная | Генерировать больше из этой категории |
| `easy_start` | Быстрый старт | MVP простой → приоритет простым идеям |
| `clear_audience` | Понятно кому | Аудитория конкретная → так и продолжать |
| `good_money` | Деньги понятны | Монетизация прозрачная → важный критерий |

## UX-флоу
```
Карточка идеи
  ├─ Нажал "Интересно" → выехали 4 кнопки "Чем?" → клик (мультиселект) → готово
  ├─ Нажал "Отбросить" → выехали 5 кнопок "Почему?" → клик → готово
  └─ 💬 → текстовое поле (опционально)
```

## Хранение (Prisma)
- `rejectReason: String?` — enum-строка при отклонении
- `likeReasons: String?` — JSON-массив при одобрении (мультиселект)
- `feedbackComment: String?` — свободный текст
- Убрать `rating: Int?` (звёзды больше не нужны)

## Профиль вкуса
- Строится из 20+ оценок
- Хранится в Settings или отдельной таблице
- Формат: процентное распределение причин + текстовые инструкции для промпта
- Пересобирается при каждой генерации отчёта
