## О проекте
**1000 ботов** — система-конвейер для поиска и анализа бизнес-идей через AI-агентов.

Большое видение: управление тысячами бизнесов без сотрудников, только AI-агенты.
Сейчас: первое звено — агенты-разведчики (сбор трендов → генерация идей).

## Стек
- Next.js 16 + TypeScript + Tailwind CSS 4
- Prisma 6 + SQLite
- Claude API (Haiku) для генерации идей
- Порт: 1000

## Запуск
```bash
export PATH="$HOME/.local/node/bin:$PATH"
npm run dev
```
Открыть: http://localhost:1000

## Структура
- `src/app/` — страницы и API-роуты
- `src/lib/` — бизнес-логика (db, ai-brain, collectors)
- `src/hooks/` — хуки для API-запросов
- `src/components/` — UI-компоненты
- `prisma/` — схема БД

## Правила
- Дизайн в стиле Apple (CSS-переменные, скруглённые углы, тени)
- React.memo() на всех UI-компонентах
- Лимиты: UI < 200 строк, роуты < 300, сервисы < 400
- Не создавай файлы "на будущее"
- Проверяй `npm run build` перед заявлением о готовности

Смотри `PROGRESS.md` — там текущая фаза и что сделано.

  ## Правила авто-памяти (КРИТИЧНО — выполнять всегда)      

  Этот проект использует систему авто-памяти. Выполнять эти правила в каждой сессии.

  ### Начало сессии
  - Прочитать MEMORY.md из директории авто-памяти и все связанные файлы.
  - Коротко рассказать пользователю: статус проекта, на чём остановились в прошлый раз, какой был план,
   что дальше. Кратко — 3-5 предложений.
  - Проверить папку sessions/ в директории авто-памяти на логи старше 7 дней.
  - Если найдены — спросить: "Есть логи сессий старше 7 дней. Удалить?" Удалять только после
  подтверждения.
  - Создать (или дополнить) лог сессии: sessions/session-ГГГГ-ММ-ДД.md

  ### Сохранение в реальном времени (во время сессии)
  - Когда пользователь делится идеей, решением или планом — НЕМЕДЛЕННО записать в файлы памяти. Не
  ждать.
  - Когда задача выполнена — НЕМЕДЛЕННО обновить статус в MEMORY.md.
  - Когда принято новое направление или архитектурное решение — записать сразу.
  - Принцип: если сессия умрёт прямо сейчас, ничего важного не должно потеряться.

  ### Логи сессий
  - Записывать по ходу дела: что обсуждали, что сделали, какие идеи появились.
  - Если несколько сессий в один день — дополнять существующий файл.
  - Логи старше 7 дней — удалять только после согласия пользователя.

  ### MEMORY.md
  Держать до 200 строк. Содержит:
  - Статус проекта и на чём остановились
  - Роадмап / план развития
  - Ключевые решения
  - Ссылки на детальные файлы

  ### Детальные файлы
  - Для глубоких тем создавать отдельные файлы: architecture.md, roadmap.md, ideas.md и т.д.
  - Ссылаться на них из MEMORY.md.
  - Создавать только по необходимости, не делать пустые заглушки.
