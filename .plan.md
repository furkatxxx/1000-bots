# План: Telegram-ссылки + мобильная адаптация

## Задача
1. В Telegram-сообщении каждая идея — кликабельная ссылка на страницу идеи
2. Страница сервиса адаптирована под мобильный телефон

## Классификация: СТАНДАРТНАЯ (жёлтая)
Затрагивает ~5 файлов, есть зависимости между компонентами.

---

## Шаг 1: Добавить `siteUrl` в настройки

Нужен публичный URL сервиса (например `https://1000bots.railway.app`), чтобы Telegram мог ссылаться на страницы.

**Файлы:**
- `prisma/schema.prisma` — добавить поле `siteUrl` в Settings
- `src/app/api/settings/route.ts` — обработка нового поля
- `src/hooks/useSettings.ts` — добавить в интерфейс
- `src/app/settings/page.tsx` — поле ввода URL сервиса

## Шаг 2: Telegram — кликабельные идеи

**Файл:** `src/app/api/telegram/send-top/route.ts`

Сейчас:
```
🥇 *Название идеи* 🤖
Описание...
```

Будет:
```
🥇 [Название идеи](https://site.com/ideas/abc123) 🤖
Описание...

🔗 Все идеи → https://site.com/reports/xyz
```

Если `siteUrl` не задан — отправлять без ссылок (как сейчас).

## Шаг 3: Мобильная адаптация

Уже есть:
- ✅ Мобильный sidebar с гамбургером (lg breakpoint)
- ✅ `pt-16 lg:pt-8 lg:ml-64` в layout
- ✅ viewport meta (Next.js автоматически)
- ✅ `-webkit-tap-highlight-color: transparent`

Нужно проверить/исправить:
- `ideas/[id]/page.tsx` — метрики `grid-cols-3` могут быть слишком узкими → `grid-cols-1 sm:grid-cols-3`
- `ideas/[id]/page.tsx` — заголовок с эмодзи `flex items-start` → на маленьком экране может быть тесно
- `ideas/[id]/page.tsx` — экспертные карточки `sm:grid-cols-2` → уже ок
- `reports/[id]/page.tsx` — проверить адаптацию карточек
- `settings/page.tsx` — проверить форму на мобильных
- `globals.css` — добавить `viewport-fit=cover` для iPhone с вырезом
